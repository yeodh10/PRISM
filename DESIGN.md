# 설계 결정 기록 (Design Decisions)

PRISM의 핵심 기술 선택과 그 근거·트레이드오프. 면접에서 "왜 그렇게 했나"에 답하기 위한 정리이기도 하다.

---

## 1. 왜 RAG인가 — 법률 도메인의 환각은 신뢰 붕괴

**문제.** LLM에 "개인정보보호법 알려줘"라고 하면 그럴듯하지만 틀린 조문을 만들어낸다. 법률에서 한 글자(항·호 번호, 수치) 오류는 곧 신뢰 붕괴.

**결정.** 모델 지식이 아니라 **검색된 조문만 컨텍스트로 주입**하는 RAG. 그리고 정확성을 *프롬프트 지시*에만 맡기지 않고 **코드로 강제**한다.

- 시스템 프롬프트: 검색 조문에만 근거 / 출처 인용 / 모르면 "찾지 못함" / 면책 고지 / 현행 기준.
- 코드 후처리 `generator._finalize()`(LLM 비호출 순수 함수):
  - 인용한 조문이 실제 검색 결과에 없으면(날조 의심) **경고 배너 부착**.
  - 면책 고지가 없으면 **코드가 항상 부착**.
  - 빈 응답/`max_tokens` 중단도 사용자에게 알림.

**효과.** "유출 통지 기한"을 물으면 시행령 소관인 구체 일수를 지어내지 않고 "확인할 수 없다"고 답한다. 색인에 없는 "드론 규제"는 "찾지 못함".

**트레이드오프.** 검색이 핵심을 놓치면 "찾지 못함"이 늘 수 있다 → 데이터 커버리지(현재 37조문)와 Hybrid로 보완.

---

## 2. 왜 한국어 SBERT(로컬) — 임베딩 API 대신

**결정.** `jhgan/ko-sroberta-multitask`(768d)를 **로컬 추론**으로 사용.

**근거.** ① 한국어 법률 용어 검색 품질(영어 모델 대비 우위), ② **API 비용 0**, ③ **데이터 외부 미전송**(질의가 외부로 안 나감 — 개인정보 도메인에 부합), ④ L2 정규화 후 코사인.

**트레이드오프.** 런타임에 torch+모델(~400MB)을 메모리에 올림 → 무료 배포 티어 메모리 부담([DEPLOY.md](DEPLOY.md)에 정직하게 명시: 1GB+ 티어 또는 onnxruntime 경량화). "로컬·무료 한국어 임베딩"이라는 강점과 메모리의 교환.

---

## 3. Hybrid 검색 — 상호참조 그래프 + 외부 법령 마스킹

**결정.** 벡터 검색 top-k에 더해, 조문 본문의 **내부 상호참조를 그래프로 구조화**(`relations.py`)해 연관 조문을 함께 반환.

**핵심 버그(적대적 감사에서 발견).** "「전자정부법」 제2조"처럼 **외부 법령** 인용을 내부 제2조로 오연결하면 안 됨. 정규식으로 「…」 외부 법령 참조 구간을 마스킹하되, 연결어 처리가 과해 "「전자정부법」 제2조 **및 제15조**"에서 내부 제15조까지 삼키는 버그가 있었다 → 마스킹 꼬리 문자 클래스를 좁혀 수정. 이 불변식은 `tests/test_relations.py`로 회귀 방지.

---

## 4. 모델·파라미터 — claude-opus-4-8 / adaptive thinking / effort

- **모델**: `claude-opus-4-8`(한국어 품질·instruction following). 비용 민감 데모는 `.env`의 `CLAUDE_MODEL`로 sonnet/haiku 교체 가능.
- **thinking**: `{"type":"adaptive"}` — 근거 추론에 유리. 깊이는 `output_config.effort`로 조절(env `CLAUDE_EFFORT`, low|medium|high|xhigh|max).
- **max_tokens=16000**: adaptive thinking이 같은 예산을 소비하므로, 답변이 thinking에 토큰을 뺏겨 잘리는 것을 방지하려 상향.
- **비스트리밍**: 단발 Q&A라 단순성을 택함(스트리밍은 향후 UX 개선 항목).

---

## 5. 데이터 정확성(날조 0) 파이프라인

**원칙.** 법조문 본문은 내 기억이 아니라 **국가법령정보센터(law.go.kr) 원문**에서.

**방법.** 조문별 `print` URL 원문 수집 → **병렬 에이전트**로 모으고 → **독립 재페치로 교차검증**(서로 다른 출처가 호 개수·중간점 "ㆍ"까지 일치하는지). 현행 버전(법률 제20897호, 시행 2025-10-02) 명시. 현재 **37조문**.

**사례.** 과징금은 추측성 "10%"가 아닌 **현행 3%**(제64조의2). 제33조는 "안전조치"가 아니라 **"개인정보 영향평가"**임을 검증 단계에서 잡아 분류 수정(실제 안전조치의무는 제29조).

---

## 6. 적대적 전수조사 → 코드 강제 안전망

멀티에이전트로 자체 시스템을 적대적으로 감사 → **"출처 강제가 시스템 프롬프트 지시일 뿐 코드 보장이 없다"**는 갭을 스스로 발견. 이를 결정론적 후처리(`_finalize`)로 코드 강제하고 순수 함수로 분리해 **pytest로 회귀 방지**. 데이터 커버리지 갭(끊긴 상호참조)·뉴스 fetch 보안도 함께 도출·수정.

---

## 7. 보안 하드닝

- **뉴스 fetch SSRF 방어**: 사설/루프백/링크로컬/**클라우드 메타데이터 IP**(169.254.169.254) 차단, 리다이렉트 각 홉 재검증, 응답 2MB 상한.
- **캐시 단일비행(single-flight) 락**: 동시 요청 시 외부 호출 중복(스탬피드) 방지.
- **레이트리밋**(IP당 분당 한도), **CORS 설정화**, `/health` 인덱스 카운트 점검.
- 법령 답변과 뉴스(외부 정보)는 명확히 분리 — 뉴스는 참고용임을 UI·응답에 표기.

---

## 8. 실전 디버깅 — Windows Smart App Control × scipy

이 개발 PC는 SAC(Enforce)가 켜져 있어 pip로 받은 **scipy ≥1.15의 `_cyutility` 컴파일 DLL을 차단** → sklearn→sentence-transformers 연쇄 import가 실패. 모듈별로 영향 범위를 진단(torch·chromadb는 통과)하고, 해당 DLL이 없는 **scipy<1.15(1.14.x)로 고정**해 numpy 2.x 호환을 유지하며 해결. (배포용 Linux에선 불필요하지만 무해.)

> 같은 SAC 환경에서 Playwright/Chromium은 통과해 스크린샷 자동화에 사용했다.

---

## 9. 테스트 전략 — LLM을 빼고 불변식을 검증

LLM 호출은 비결정적·유료라 CI에 부적합. 그래서 **결정론적 안전 불변식만** 분리해 테스트한다(`pytest`, LLM 비호출):

- 면책 고지 강제, 출처 화이트리스트(미검증 인용 경고), 외부 법령 마스킹, 뉴스 SSRF 호스트 판정, `/ask` 입력 검증(422)·키 없음(503), `/health`.
- 이를 위해 `_finalize`를 generate에서 **순수 함수로 분리**한 것이 핵심(테스트 가능성 설계).
- GitHub Actions CI: `validate_data` + `pytest`.

## 부록 — 예상 면접 질문 & 한 줄 답변

- **환각을 어떻게 막았나?** 검색된 조문만 컨텍스트로 주입 + 출처·고지를 코드(`_finalize`)로 강제, 미검증 인용엔 경고 배너. 모르면 "찾지 못함".
- **왜 로컬 임베딩?** 비용 0·질의 데이터 외부 미전송·한국어 품질. 트레이드오프는 런타임 메모리(배포 티어/ONNX로 대응).
- **Hybrid가 뭘 더 주나?** 조문 상호참조 그래프로 연관 조문까지. 외부 법령 인용은 마스킹해 오연결 방지.
- **정확성을 어떻게 보장?** law.go.kr 원문 수집 + 독립 재페치 교차검증, 그리고 코드 불변식을 pytest로.
- **가장 어려웠던 문제?** ① SAC의 DLL 차단을 모듈 단위로 진단해 scipy 핀으로 해결, ② 적대적 자가 감사로 "출처 강제가 코드 보장이 아니던" 갭을 찾아 `_finalize`로 코드화.
- **확장한다면?** 시행령·해설서 데이터, 검색 정확도 eval(recall@k) 상시화, 스트리밍 응답, 임베딩 ONNX 경량화 배포.
